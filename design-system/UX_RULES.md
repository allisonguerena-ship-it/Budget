# UX Rules

## Primary user goal

The user wants to quickly understand:

1. What can I spend this week?
2. What did I spend?
3. What is left over?
4. How does that affect my POT?
5. Are future bills/trips/paydays going to make cash tight?

## Keep the app emotionally simple

Finance apps can feel stressful. This app should feel like a calm personal control center.

## Avoid clutter

Do not show every advanced setting at once.

Use collapsible sections for:

- Fixed bills
- Data tools
- Long settings
- Advanced cash-flow constants

## Make time logic explicit

The user cares that past weeks are not falsely rewritten.

Always show when a week is finalized/frozen.

Use labels like:

- Finalized
- Open
- Frozen on [date]
- Fixed bills used
- POT after week

## Make editability obvious

Locked fields should look disabled.

Finalized weeks should disable row editing.

Cash-flow constants should be locked by default.

## Mobile behavior

On mobile:

- One column
- Large tap targets
- Weekly cards stacked
- Tables should scroll horizontally if needed
- Avoid tiny controls

## Avoid misleading projections

If the cash-flow planner has no starting cash, say so clearly.

Do not pretend the projection is accurate when starting cash is zero.

## Wording

Use plain-language finance wording.

Prefer:

- Left over
- Fixed bills
- Planned expenses
- Paydays
- Credit card due
- Safety buffer

Avoid:

- Liabilities
- Forecasting engine
- Allocation model
- Obligations module
