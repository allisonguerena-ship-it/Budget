# Design Principles

POT Budget should feel calm, clean, and trustworthy.

The design should feel closer to a simple Apple-style personal app than a corporate dashboard or spreadsheet.

## Visual direction

Use:

- Cream / soft neutral backgrounds
- Soft pink
- Sage green
- Warm gold
- Rounded cards
- Clear spacing
- Large readable numbers
- Minimal clutter

Avoid:

- Harsh colors
- Dense tables everywhere
- Too many charts
- Tiny controls
- Complicated filters
- Enterprise dashboard styling

## Information hierarchy

The most important information should be visible first:

1. Weekly allowance
2. Spent
3. Remaining
4. POT
5. Fixed bills
6. Cash-flow projection
7. Planned expenses

The user should not have to search for the number that matters.

## Interaction style

The app should feel:

- Fast
- Calm
- Forgiving
- Easy to edit
- Easy to understand

When possible, use:

- Cards
- Collapsible sections
- Clear labels
- Helpful empty states
- Simple confirmations for risky actions

## Editing philosophy

Do not redesign the whole app unless explicitly requested.

Prefer small UI improvements that preserve the existing feel.
