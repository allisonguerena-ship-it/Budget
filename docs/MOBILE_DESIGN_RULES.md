# Mobile Design Rules

Mobile is the priority.

The app should work well on iPhone in portrait orientation.

## Layout rules

Use:

- Single-column layouts
- Stacked cards
- Collapsible sections
- Large tap targets
- Comfortable spacing

Avoid:

- Wide tables
- Tiny text
- Tiny buttons
- Required horizontal scrolling
- Layouts that require zooming

## Touch rules

Tap targets should be comfortable for thumbs.

Important actions should be reachable without precision tapping.

Inputs should be easy to edit on a phone.

## Budget UI rules

On mobile:

- Weekly summary cards should be easy to scan.
- Week tabs should scroll cleanly or convert to a dropdown.
- Fixed bills should stay collapsed unless opened.
- Cash-flow planner should not require zooming.
- Expense rows may become stacked cards if tables are too cramped.

## Mobile testing checklist

After UI changes, test:

- iPhone-sized width
- Long fixed bill names
- Many fixed bills
- Many weekly expenses
- Keyboard open while editing inputs
- Cash-flow section
- Finalized week controls
