# Feature Decision Framework

Before adding any feature, ask these questions.

## Must-answer questions

1. Does this help answer: “How much money can I safely spend?”
2. Will it be used weekly or often?
3. Does it preserve the core budgeting logic?
4. Does it make the app easier instead of more confusing?
5. Can it stay hidden unless needed?
6. Does it work on iPhone?
7. Does it protect existing data?

If the answer is mostly no, do not build it.

## Prefer

Prefer features that:

- Reduce confusion
- Save time
- Prevent mistakes
- Protect data
- Improve mobile usability
- Support multi-device sync

## Avoid

Avoid features that:

- Add clutter
- Duplicate existing sections
- Turn the app into a generic finance dashboard
- Require too many settings
- Make the user maintain extra data
- Blur the line between POT and cash flow

## Feature priority order

1. Data safety
2. Core budget correctness
3. Cloud sync stability
4. Backup/import/export
5. Mobile usability
6. Small UI polish
7. Nice-to-have analytics
